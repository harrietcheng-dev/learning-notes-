# 📅 课表功能开发总结（Schedule Feature）

## 一、项目背景

在移动端 App 中实现“我的课表”功能，需要从学校教务系统获取课程数据，并在客户端进行解析和展示。

由于教务系统未提供标准 API，因此采用 WebView + JS 抓取的方式获取数据。

---

## 二、问题记录

### ❗ 教务系统无法打开

在开发初期，遇到 WebView 无法正常加载教务系统页面的问题。

**排查过程：**
- 检查网络连接 ✅
- 检查 URL 是否正确 ✅
- 检查 WebView 权限（如网络权限）✅
- 最终发现是：
  - 教务系统存在访问限制 / 登录状态问题
  - 需要正确进入登录态页面后才能加载数据

**解决方案：**
- 确保 WebView 已完成登录流程
- 在页面加载完成后再执行数据抓取

---

## 三、核心实现流程

整体数据流如下：
教务系统 → WebView → JS抓取 → JSON → 解析 → UI展示

### 1️⃣ WebView 加载教务系统

- 使用 WebView 打开教务系统页面
- 用户完成登录
- 进入课表页面

---

### 2️⃣ JS 抓取页面数据

通过注入 JavaScript，从页面中提取课程数据，例如：

- 课程名称（KCMC）
- 上课时间（KSJC / JSJC）
- 上课地点（JASMC）
- 教师信息（XSKJS）

👉 示例字段说明：

| 字段 | 含义 |
|------|------|
| KCMC | 课程名称 |
| JASMC | 教室名称（上课地点） |
| KSJC | 开始节次 |
| JSJC | 结束节次 |
| XSKJS | 教师信息 |

---

### 3️⃣ 转换为 JSON 数据

JS 抓取到的数据整理为 JSON 格式，例如：

```json
{
  "KCMC": "操作系统原理",
  "JASMC": "前卫-法学楼-304",
  "KSJC": 3,
  "JSJC": 5,
  "XSKJS": "宋姗姗"
}
